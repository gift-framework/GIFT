Section 4: CONSTRUCTION MÉTRIQUE DE HITCHIN (Cell 9)
----------------------------------------------------
Fonction: metric_from_phi_hitchin()

Théorie:
B(u,v) = (i_u φ) ∧ (i_v φ) ∧ φ (forme bilinéaire)

Algorithme itératif:
1. Départ: g₀ = I (euclidien)
2. Calcul ψ = *_g φ
3. Pour vecteurs de base e_i: s_ij = *(e_i ⌟ φ) ∧ (e_j ⌟ φ) ∧ φ
4. Construction métrique g depuis s_ij
5. Normalisation det(g) = 1
6. Itération jusqu'à convergence

Paramètres:
- n_iter: 5 (défaut) ou curriculum 1→2→3
- g_init: warm start optionnel
- Sortie: metric (batch, 7, 7) SPD

Références:
- Hitchin (2000) "Stable forms and special metrics"
- Bryant "Metrics with exceptional holonomy"

Section 5: CALCUL DE TORSION (Cell 11)
---------------------------------------
Fonction: compute_torsion_full()

Décomposition de Fernández-Gray:
- τ₀: dilaton (scalaire)
- τ₁: 1-forme (7 composantes)
- τ₂: 2-forme (21 composantes)
- τ₃: 3-forme (35 composantes)

Équations:
- dφ = τ₀ ψ + 3 τ₁ ∧ φ + *τ₃
- d*φ = 4 τ₁ ∧ *φ + τ₂ ∧ φ

Condition torsion-free:
- dφ = 0 AND d*φ = 0

Retourne:
{
  'dphi': 4-forme dφ
  'dstar_phi': 3-forme d*φ
  'tau0': scalaire
  'tau1': 1-forme
  'tau2': 2-forme
  'tau3': 3-forme
  'torsion_norm': norme totale
}

Section 9: FONCTIONS DE PERTE (Cell 20)
----------------------------------------
1. hitchin_functional(phi, metric)
   H(φ) = ∫ √det(B(φ)) dvol
   
   Implémentation:
   - volume_penalty: (√det(g) - 1)²
   - anisotropy: max(λ)/min(λ) des valeurs propres
   - Retour: volume_penalty + 0.1·log(anisotropy)

2. combined_loss(phi, coords, metric, h2_forms, torsion_dict)
   Composantes:
   - Torsion: ||dφ||² + ||d*φ||² (poids: 2.0)
   - Hitchin: H(φ) (poids: 1.0)
   - Closure: ||dω||² (poids: 0.5)
   - Orthogonality: ||Gram(H²) - I|| (poids: 0.3)
   
   Total = 2·torsion + 1·hitchin + 0.5·closure + 0.3·ortho

Section 11: BOUCLE D'ENTRAÎNEMENT (Cell 27)
--------------------------------------------
Features clés:

1. AUTO-RESUME:
   - Détecte dernier checkpoint
   - Continue depuis epoch sauvegardée

2. BOUCLE PRINCIPALE:
   for epoch in range(start_epoch, epochs):
     - Échantillonnage région selon curriculum
     - Gradient accumulation (2 steps)
     - Mixed precision (AMP)
     
     Forward:
       phi = phi_network(coords)
       metric = metric_from_phi_hitchin(phi, coords, n_iter=curriculum)
       
     Warm start:
       - g_init = metric précédente (tous les 10 epochs)
       
     Torsion subsampling:
       - Si batch > 256: sous-échantillonner 256 points
       - Recalcul phi sur sous-ensemble (préserve autograd)
       
     Backward:
       - scaler.scale(loss).backward()
       - Gradient clipping: max_norm=0.5
       - scaler.step(optimizer)

3. LOGGING (tous les 100 epochs):
   - Loss, torsion, hitchin, closure, ortho
   - Temps écoulé et ETA
   - Curriculum phase actuelle

4. CHECKPOINTS (tous les 1000 epochs):
   - Sauvegarde modèles + optimiseur + scheduler + history
   - Copie vers Google Drive

5. EARLY STOPPING:
   - Epoch > 3000 AND torsion < 1e-9 AND closure < 1e-3

6. SAUVEGARDE FINALE:
   - phi_network_final.pt
   - harmonic_network_final.pt

MATHÉMATIQUES RIGOUREUSES v0.8b
================================================================================

1. CONSTRUCTION HITCHIN:
   - Métrique depuis 3-forme stable
   - Itération B(u,v) = (i_u φ) ∧ (i_v φ) ∧ φ
   - Normalisation det(g) = 1

2. TORSION EXPLICITE:
   - dφ calculé via autograd
   - Décomposition τ₀, τ₁, τ₂, τ₃
   - Loss: ||dφ||² + ||d*φ||² → 0

3. HODGE STAR MÉTRIQUE:
   - *_g dépend de g courant
   - Produits L²: ∫ α ∧ *_g β
   - Gram matrices avec métrique

4. FORMES HARMONIQUES:
   - Closure: dω = 0 (via loss)
   - Coclosure: d*ω = 0
   - Indépendance linéaire modulo exact

5. TCS BOUNDARY:
   - φ → φ_std cylindrique sur S¹×K3
   - Rotation gluing ξ
   - Identifications périodiques

6. FONCTIONNELLE HITCHIN:
   - H(φ) = ∫ √det(B(φ)) dvol
   - Minimisation → G₂ torsion-free
   - Objectif principal d'entraînement
